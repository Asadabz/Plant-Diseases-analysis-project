# Leaf Scan — Plant Disease Detector

End-to-end ML project: upload a leaf photo → get a disease diagnosis with confidence score.
Transfer learning (MobileNetV2) + FastAPI backend + React frontend + Docker.

```
plant-disease-detector/
├── training/        # train the model (run on Colab or local GPU)
│   ├── train.py
│   └── requirements.txt
├── backend/          # FastAPI inference API
│   ├── main.py
│   ├── requirements.txt
│   ├── Dockerfile
│   └── model/        # put plant_disease_model.h5 + class_names.json here
├── frontend/         # FloraScan — React + Vite + Tailwind UI (3 pages: Home, Library, About)
│   ├── src/
│   │   ├── pages/        # home.tsx (scan+diagnose), library.tsx (disease encyclopedia), about.tsx
│   │   ├── lib/detection.ts  # calls the real backend /predict and enriches the result
│   │   └── components/   # shadcn/ui component set + layout/Sidebar
│   ├── Dockerfile
│   └── package.json
└── docker-compose.yml
```

This is a complete, working skeleton — the API and frontend run and have been tested end-to-end
(backend boots and serves `/health`, frontend builds cleanly, type-checks with zero errors, and
serves all three routes). The one thing you need to add yourself is a **real trained model**, since
that needs a real dataset and (ideally) a GPU. Steps below.

**Quick test first (no training needed):** A placeholder model is already included at
`backend/model/plant_disease_model.h5`, generated by `backend/create_dummy_model.py`. It has only
ever seen random noise, so its predictions are meaningless — but it lets you confirm the full
pipeline (upload → API → response → UI) works before you spend hours on real training. Run it as-is
(Step 2 or 3 below) and you should see a (random, wrong) diagnosis appear instead of a "model not
loaded" error. Once you're ready for real predictions, just train a real model (Step 1) and replace
the two files in `backend/model/`.

**About the frontend:** it's a polished multi-page UI ("FloraScan") with a Home page (drag-and-drop
leaf scanner with a live progress animation and a tabbed result panel — diagnosis / treatment /
prevention), a Library page (a searchable encyclopedia of the diseases the model can recognize),
and an About page. `src/lib/detection.ts` calls your FastAPI backend's `/predict` endpoint and maps
the raw model class name (e.g. `Tomato___Late_blight`) to rich info — symptoms, treatment,
prevention — using a built-in disease knowledge base, with a generic fallback for any class it
doesn't recognize.

## Step 1 — Train the model (do this on Google Colab, it's free and has a GPU)

1. Get the dataset: [PlantVillage on Kaggle](https://www.kaggle.com/datasets/emmarex/plantdisease)
   (or any dataset organized the same way — one subfolder per disease class).
2. Open a new Colab notebook, set runtime to **GPU** (Runtime → Change runtime type → T4 GPU).
3. Upload `training/train.py` and the dataset (via `kaggle.json` API key, or Google Drive).
4. Arrange the dataset like this:
   ```
   dataset/
       train/
           Tomato___Healthy/...
           Tomato___Late_Blight/...
       val/
           Tomato___Healthy/...
           Tomato___Late_Blight/...
   ```
   (Kaggle's PlantVillage download is one big folder per class — split ~80/20 into train/val
   first; a one-line script with `shutil` and `random.sample` does this.)
5. Run:
   ```
   !pip install -q -r requirements.txt
   !python train.py --data_dir dataset --epochs 10
   ```
6. Download the two output files from Colab:
   - `model/plant_disease_model.h5`
   - `model/class_names.json`
7. Place both files into `backend/model/` in this project.

Training ~10 epochs on a Colab T4 GPU usually takes 15–40 minutes depending on dataset size,
since MobileNetV2's pretrained layers are frozen (only the small classifier head trains).

## Step 2 — Run everything locally (no Docker, fastest for development)

**Backend:**
```bash
cd backend
pip install -r requirements.txt
uvicorn main:app --reload --port 8000
```
Visit `http://localhost:8000/health` — should show `{"status":"ok","model_loaded":true}`.

**Frontend:**
```bash
cd frontend
npm install
npm run dev
```
Visit `http://localhost:5173` — Home page lets you upload/drop a leaf photo and run a live
diagnosis; Library page shows the disease encyclopedia; About explains the project.

---

## Step 3 — Run everything with Docker (closer to production)

From the project root:
```bash
docker compose up --build
```
- Frontend: `http://localhost:5173`
- Backend API: `http://localhost:8000`

The backend container mounts `backend/model/` as a volume, so you can swap in a newly
trained model without rebuilding the image — just restart the container.

## Step 4 — Deploy for free

| Platform | Notes |
|---|---|
| **Hugging Face Spaces** | Best for ML — supports Docker Spaces directly, generous free tier, occasional free GPU |
| **Render** | Free web service tier works for backend; keep the model small (MobileNetV2 is ~14 MB) |
| **Railway** | Similar to Render, easy Docker deploys |

Deploy `backend/` and `frontend/` as two separate services, and point the frontend's
`VITE_API_URL` build arg at the backend's public URL.

## API reference

**`GET /health`**
```json
{ "status": "ok", "model_loaded": true }
```

**`POST /predict`** — multipart form, field name `file`
```json
{
  "disease": "Tomato___Late_blight",
  "confidence": 0.94,
  "top3": [
    { "disease": "Tomato___Late_blight", "confidence": 0.94 },
    { "disease": "Tomato___Early_blight", "confidence": 0.04 },
    { "disease": "Tomato___Healthy", "confidence": 0.01 }
  ]
}
```
